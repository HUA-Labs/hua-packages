# 용어집 (Glossary)

> Sum Diary 프로젝트 핵심 용어 정의

---

## 프로젝트 용어

### HUA (후아)

- **정의**: 프로젝트/조직 이름. 감정 분석 엔진의 브랜드
- **맥락**: HUA AI Service는 2차 감정 분석을 수행하는 엔진
- **HUA 감정 엔진**: 1차 AI 분석 결과를 기반으로 정량 지표(valence, arousal, entropy, density)를 생성하는 2차 분석 시스템

### Sum Diary (숨다이어리)

- **정의**: HUA 프로젝트의 감정 일기 서비스
- **이름 유래**: "숨" = 호흡(breath) + 숨기다(hide)의 중의적 의미
- **핵심**: Privacy-First AI 감정 일기

### 숨다 (Sumda)

- **정의**: AI 캐릭터 이름. 사용자에게 공감하는 분석가 페르소나
- **성격**: 공감적, 비판단적, 따뜻한 어조
- **역할**: 일기 분석 결과를 사용자에게 전달하는 인격화된 AI

## 분석 용어

### Slip

- **정의**: AI 과몰입(emotional over-engagement) 상태를 나타내는 지표
- **레벨**:
  - `none` - 정상
  - `soft` - 경미한 과몰입 (tier_m >= 4.0, tier_a <= 2.5)
  - `hard` - 심각한 과몰입 (tier_m >= 4.5, tier_a <= 2.0)
- **의미**: 감정 강도(tier_a)가 낮은데 서사 역동성(tier_m)이 높을 때 발생. 사용자가 감정을 느끼지 않으면서 AI에 과몰입하는 패턴

### Tier A (Affect Tier)

- **정의**: 감정 강도 지표 (1.0 ~ 5.0)
- **의미**: 일기에 표현된 감정의 깊이/강도
- **예시**: 1.0 = 감정 표현 거의 없음, 5.0 = 매우 강한 감정

### Tier M (Momentum Tier)

- **정의**: 서사 역동성 지표 (1.0 ~ 5.0)
- **의미**: 일기의 서사적 움직임/전개 정도
- **예시**: 1.0 = 단순 기록, 5.0 = 극적인 서사 전개

### Valence

- **정의**: 감정 극성 (-1.0 ~ 1.0)
- **범위**: -1.0 = 매우 부정적, 0 = 중립, 1.0 = 매우 긍정적
- **소스**: HuaEmotionAnalysis (2차 분석)

### Arousal

- **정의**: 각성 수준 (0.0 ~ 1.0)
- **범위**: 0.0 = 매우 낮은 각성(졸림/무기력), 1.0 = 매우 높은 각성(흥분/긴장)
- **소스**: HuaEmotionAnalysis (2차 분석)

### Entropy

- **정의**: 감정 복잡도 (0.0 ~ 1.0)
- **범위**: 0.0 = 단일 감정, 1.0 = 매우 복잡한 감정 혼재
- **소스**: HuaEmotionAnalysis (2차 분석)

### Density

- **정의**: 감정 밀도 (0.0 ~ 1.0)
- **범위**: 0.0 = 감정 표현 희박, 1.0 = 감정 표현 밀집
- **소스**: HuaEmotionAnalysis (2차 분석)

### Ethics Tags

- **정의**: AI가 일기에서 감지한 윤리적/감정적 태그 목록
- **예시**: `hope`, `anxiety`, `grief`, `resilience`, `isolation`
- **용도**: Slip 계산 보정, 태그 빈도 분석

## 위기 감지 용어

### Crisis Detection (위기 감지)

- **정의**: 일기에서 위기 신호(자살, 자해 등)를 감지하는 3단계 시스템
- **3단계**:
  1. AI 감지 - GPT/Gemini가 ethics 태그로 감지
  2. 키워드 Fail-Safe - AI 누락 대비 키워드 매칭
  3. 운영자 최종 판단 - 관리자 대시보드 검토

### CrisisType (위기 타입)

| 타입 | 설명 |
|------|------|
| `SUICIDE` | 자살 위험 |
| `SELF_HARM` | 자해 위험 |
| `DRUG` | 약물 남용 |
| `CHILD_ABUSE` | 아동 학대 |
| `SERIOUS_MEDICAL` | 심각한 의료 상황 |
| `TERRORISM` | 테러/폭력 |

### Risk Level (위험도 레벨)

| 레벨 | 의미 |
|------|------|
| 0 | 없음 |
| 1 | 낮음 |
| 2 | 중간 |
| 3 | 높음 |
| 4 | 위기 (즉시 개입 필요) |

### Escalation (에스컬레이션)

- **정의**: 히스토리 기반 위험도 상승 메커니즘
- **조건**:
  - 최근 7일 3건 이상 → +1
  - 연속 3일 이상 → +1
  - 평균 위험도 높고 반복 → +1

## 일기 용어

### 일기 타입

| 타입 | 조건 | 분석 | 수정 |
|------|------|------|------|
| 오늘 일기 | `diary_date === today` | 가능 | 불가 |
| 나중 일기 | `diary_date < today`, 24시간 이상 지연 | 가능 | 불가 |
| 미래 일기 | `diary_date > today` | 없음 | 가능 |

### Delayed Entry (나중 일기)

- **정의**: 과거 날짜에 대해 작성한 일기
- **조건**: `actual_written_at - diary_date > 24시간`
- **플래그**: `is_delayed_entry = true`
- **분석**: 가능하지만 수정 불가

### Soft Delete (소프트 삭제)

- **정의**: 데이터를 실제 삭제하지 않고 `deleted_at` 타임스탬프 설정
- **복구**: `deleted_at = null`로 설정하여 복구 가능
- **조회**: `deleted_at IS NULL` 조건으로 필터링

## 기술 용어

### SSE (Server-Sent Events)

- **정의**: 서버에서 클라이언트로 실시간 데이터를 단방향 전송하는 기술
- **용도**: 일기 분석 결과를 섹션별로 실시간 전송
- **흐름**: title → summary → emotion_flow → question → interpretation

### 1차 분석 / 2차 분석

| 구분 | 1차 분석 | 2차 분석 |
|------|----------|----------|
| 용도 | 사용자 화면 | 데이터 분석 |
| 프로바이더 | OpenAI/Gemini | HUA 엔진 (Gemini) |
| 입력 | 원문 사용 | 익명화된 텍스트 |
| 출력 | 제목, 요약, 감정 흐름 등 | valence, arousal, entropy 등 |
| 타이밍 | 동기 (SSE) | 비동기 |

### Privacy-First

- **정의**: 개인정보 보호를 최우선으로 하는 설계 원칙
- **구현**:
  - 일기 내용 AES-256-GCM 암호화
  - 개인정보 해시 기반 검색
  - 2차 분석은 익명화된 텍스트만 사용

## 인프라 용어

### Doppler

- **정의**: 시크릿/환경변수 관리 서비스
- **용도**: API 키, DB URL 등 민감한 환경변수를 안전하게 관리
- **사용**: `doppler run -- next dev` 형태로 환경변수 주입

### Multi-Schema (Prisma)

- **정의**: 하나의 PostgreSQL DB에서 여러 스키마 사용
- **구조**:
  - `user` 스키마: 사용자/일기/분석 데이터
  - `admin` 스키마: 위기알림/감사로그/운영 데이터
- **이유**: 데이터 격리, 접근 제어

## 결제 용어

### 토스페이먼트

- **정의**: 한국 결제 서비스 (Toss Payments)
- **연동 모델**: Subscription, Payment, PaymentMethod
- **키**:
  - `paymentKey` - 결제 키 (결제 승인 후 발급)
  - `billingKey` - 빌링 키 (자동결제용)
  - `customerKey` - 고객 키
